Mission draft · AI Control Center

The human cockpit for agentic repositories.

Markdown stays the source of truth for agents. The Control Center becomes the layer where humans can see the system, trust the work, and decide what happens next.

North star

Open any repo and know what the AI system knows, what it is allowed to do, what changed, and what needs your attention.

Not another dashboard. A repo-native trust surface: docs, agents, rules, decisions, commands, hooks, TODOs, risks, and evidence in one inert HTML file.
Not a second source of truth. The HTML reflects markdown. It never replaces it. It makes the machine-readable world human-readable.
Not decoration. Every card should answer: what is here, why does it matter, what changed, and what should I do?
Mission

Make AI-heavy repos legible, governable, and alive.

Agentic software projects are becoming small operating systems: provider folders, prompts, skills, commands, MCP config, rules, specs, plans, receipts, reviews, and handoffs. The code may be clean, but the AI layer becomes invisible fast.

AI Control Center should be the first file a human opens when they inherit, review, or operate an AI-native repo. It answers the orientation questions before anyone has to grep.

Vision

From inventory to command room.

Phase 3 makes the repo visible. Later phases can make it directional: “what changed since last stamp?”, “which rule owns this behavior?”, “what evidence proves this gate?”, “which agent should touch this next?”

The best version feels like a mission control room for the repo’s intelligence layer — calm, grounded, source-linked, and impossible to bluff.

How to make it better

Five upgrades that turn the idea into a category.

01 · Orientation

Start with “what is this repo?”

Overview should summarize purpose, providers detected, AI surfaces, open risks, and freshest docs. No fake metrics. Only real data.

02 · Change awareness

Show what moved.

A future stamp diff can reveal new rules, removed commands, changed MCP servers, stale specs, and newly-open TODOs.

03 · Trust posture

Surface evidence, not vibes.

Redaction status, generation digest, schema version, source paths, and validation warnings should be visible without making the UI scary.

04 · Relationship map

Connect the pieces.

Rules point to plans. Plans point to tests. Commands point to scripts. Agents point to permissions. The map is the repo’s nervous system.

05 · Action handoff

Help the next agent start well.

Each section can suggest the safest next prompt: “review this rule,” “run this verification,” “update this stale plan,” “ask about this open decision.”

06 · Code-oz bridge

Make gates visible.

For Code-oz repos, show phase state, gates, approvals, review artifacts, event ledger health, and the exact evidence behind “ready.”

Product posture

Calm authority beats flashy intelligence.

1

Everything links back to source.

If a card cannot link to a file, it should probably not exist yet.

2

The UI never invents certainty.

Unknown is a valid state. Stale is a valid state. Missing is a valid state.

3

Humans should leave smarter.

Every view should reduce the cost of understanding the repo by one meaningful step.

Manifesto

AI Control Center is for the moment when a repo stops being “just code.”

Modern repos carry instructions, roles, rituals, memory, policy, and operating constraints. Agents can parse that maze. Humans need a cockpit.

The winning version is not the most animated or the most complex. It is the one that makes trust visible. It lets Ozzy, Hermes, Claude, Codex, Cursor, and Code-oz all work from the same map without pretending the HTML is the truth. The truth stays in the files. The Control Center makes the truth usable.
